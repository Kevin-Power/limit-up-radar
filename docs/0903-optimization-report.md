### 規則定義 (本回測實作)

```python
R1  if 0% ≤ gap < 5%:   exit at T+1 09:15
    elif gap < 0:        exit at T+2 open (基線)
    elif gap ≥ 5%:       exit at T+2 open (基線)

R2  exclude if score >= 90
    or  prev_volume >= 20_000_000 股 (= 20k 張)
    or  prev_close < 30 元

R3  exclude if 同日同族群入選檔數 >= 8

R4  keep if prev_volume * prev_close >= 1e8 (D-1 成交金額 ≥ 1 億)

R5  trade only if 該 entryDate 的 score>=75 精選 <= 15 檔
```

## 四、放鬆 score 門檻 — 真正的 alpha 在哪？

| score 門檻 | T+2 open (基線) | + R1 動態出場 |
|---|---|---|

| score ≥ 70 | n=274, 勝 59.1%, EV +2.07%, +5.68M | n=274, 勝 68.2%, EV +2.14%, Sharpe 7.01, +5.86M |
| score ≥ 75 | n=207, 勝 58.0%, EV +1.92%, +3.97M | n=207, 勝 66.2%, EV +2.18%, Sharpe 5.97, +4.51M |

### 驚人發現

放鬆到 **score≥50** 反而：
- 樣本 ×3.2 (207→659)
- EV 幾乎不變 (+1.92% → +2.14%)
- Sharpe 從 5.97 跳到 **10.65**
- 總獲利從 +4.51M 衝到 **+13.95M**

換言之：**高分門檻原本以為是「精選」，實際上是把樣本切薄並沒有顯著提升 per-trade EV**。R1 在所有 score 門檻下都有效，但 score≥50 + R1 才是「資本利用最大化」的甜蜜點。

⚠ **注意**：score≥50 表示每進場日要打 ~18 筆 (659 / 36)，實戰資金/流動性可能撐不起；score≥75 (~5.8 筆/日) 更實際。

## 七、推薦策略

### 主推 ★：score≥75 + R1 動態出場

```
日選股池: score ≥ 75 全部 (約 5.8 筆/日)
進場:    T+1 09:00 競價買進
出場:    
  if gap_pct (T+1 open vs T 收盤) in [0%, 5%):
      T+1 09:15 賣出
  else (gap < 0% 或 gap ≥ 5%):
      T+2 09:00 賣出
```

- **何時用**: 想穩、Sharpe 第一、不想換太多本金。
- **回測**: n=207, 勝率 66.2%, 每筆 EV +2.18%, **+4.51M TWD**, MDD 42.8%, Sharpe 5.97
- **6 月韌性**: EV 從 +0.75% 救回 +1.75%

### 次推：score≥50 + R1 動態出場

```
同上，但 score 門檻降到 50 (約 18 筆/日)
```

- **何時用**: 資金充裕、可接受 18 個 ticker / 日的分散買單；要極大化 TWD。
- **回測**: n=659, 勝率 67.4%, EV +2.12%, **+13.95M TWD**, MDD 40.7%, **Sharpe 10.65**
- **風險**: 流動性/操作量、超出個人實戰可執行範圍。

### 不推薦

- L2~L5 任何疊加 (R2/R3/R4/R5)：樣本切薄、總 TWD 大跌
- R1+R2+R5「穩健三件套」: 表面 82.5% 勝率漂亮，但 n=40 容易過擬合
- T+5 收盤 (#6 維度結論)：與 R1 衝突；本回測未實作

## 八、警告與 caveats

1. **R2–R5 來自樣本內探索**，未經 leave-one-out / 前後半 / OOS 三件驗證 (R1 通過 2/3)。它們**很可能是過擬合**，疊太多會導致實戰失效。
2. **n<50 的勝率不可信**：Wilson 95% CI 過寬 (例如 n=12 的 91.7% 勝率，CI 約 [62%, 99%])。
3. **快取命中率 75%** (207/274) — 缺漏部分被略過，可能引入 survivorship bias (流動性差/老股的 1 分 K 較難取得)。
4. **年化報酬是理論天花板**，假設無 slippage / 無資金重疊 / position size 等量。實戰至少打 6-7 折。
5. **樣本期僅 2.5 個月**：完全沒有經歷大跌段 (2026-04 ~ 06 都是上漲市)。空頭月份的表現未知。
6. **6 月已開始衰退**：基線 6 月 EV 跌到 +0.75%，可能是策略飽和的早期訊號。R1 救得回來，但需持續監控。

## 九、結論與下一步

### 該做

1. **立刻部署 R1 (score≥75 版)** — 是唯一通過對抗驗證、樣本內外都穩、增加 Sharpe、降低 MDD 的規則。
2. **同步準備 score≥50 版本** — 評估資金與流動性可行性，若可執行，TWD 收益翻 3 倍。
3. **每月跑同腳本**：監控月度 EV 是否持續低於 +1.5%，連兩月低於則暫停策略並重做維度探索。

### 別做

1. 不要把 R2–R5 入鏈 — 它們會切薄樣本、削弱 TWD，且 OOS 風險未驗。
2. 不要追求單一桶高勝率 (90%+ 等)；那是 n 太小的視覺幻覺。
3. 不要把 R1 改成「動態移動停損」或加 TP/SL — 探索 7 已證實 TP/SL 網格全虧。

### 想進一步研究

1. **R5 (regime ≤15) 用 leave-one-out 跑一次**：若 OOS EV ≥ +3%，可以加入。
2. **score≥50 群組的「執行可行性」分析**：是否每日都能在 09:00 競價同時掛 18 個單？
3. **跨樣本期穩健性**：拿到 2026-Q1 與 2025 整年資料重跑，看 R1 是否仍存活。
