## 0. 一句話

把平台的對外承諾從「跟著訊號買會賺」轉成「**台股最誠實的漲停族群資料庫與隔日行為統計** — 幫你看懂市場，不是報明牌」；所有統計補上成本、分布與樣本限制，並以「誠實統計」為旗艦內容。

## 1. 為什麼轉向

- 既有頭條（勝率 7x%、平均 +4%）是**毛報酬、偏多頭小樣本、平均值**——三個方向都往好看偏。
- 對當沖型選股，勝率是最易誤導的單一指標：該看的是**扣成本後的每筆期望值與分布**（中位數 vs 平均、regime 切分）。
- 法規與信任面：分析/教育＋個人紀錄定位（非投顧、不構成建議）既誠實也安全。

## 2. 主張政策（Claim Policy）— 全站文案的硬規則

**可以說：**
- 透明統計：樣本數、毛/淨報酬、中位數、截尾平均、Wilson CI、多空分段，一律附「未含成本」或「含成本」標註。
- 「個人研究紀錄／紀律紀錄分享」「統計供研究學習」「參考區間（固定規則）」。
- 教育性判讀（為什麼平均會騙人、成本怎麼吃掉報酬、滑價為何在小型股更大）。

**不可以說（已全站清除）：**
- 任何**未含成本的數字當作可得報酬**呈現（毛數字必須標「毛」）。
- 報酬投射／複利外推（如「預估月報酬 +66%」——已刪除）。
- 指令式建議措辭當作對讀者的指示（進場/出場/目標價 → 個人紀錄、參考區間框架）。
- 把示範/模擬數據呈現得像真實統計（示範圖卡一律標「示範數據」）。

## 3. 旗艦內容：誠實期望值統計（Honest Stats）

### 3.1 引擎 `scripts/honest_stats.py`（含 `test_honest_stats.py`，pytest）
- **選股重建**：完整鏡像 `scoring.ts`（含 `run_backtest.py` 漏掉的 權值股+25／近期空吞−25）；
  趨勢族群(3日≥2)、6日窗處置/連板、營收檔按 **dataDate ≤ 選股日** 選擇（point-in-time）。
- **結算**：今日收盤買、隔日開盤賣；真實 TWSE `STOCK_DAY` 月資料，TPEx fallback 用**新端點**
  `/www/zh-tw/afterTrading/tradingStock`；空回應退避重試（TWSE 會限流，假陰性不得首輪快取）。
- **統計**：兩 cohort（top-20≥50 全樣本／高分群 ≥60）× 三成本情境（毛／費稅 0.435%／保守含滑價 1.0%）
  × {勝率+Wilson95CI、平均、中位數、截尾(去前3)、p10/25/75/90} ＋ 大盤紅/綠盤日 regime 切分 ＋ top outliers。
- **產物**：`data/analysis/honest_stats.json`（進 git；唯一數字真相，文件與 UI 不寫死數字）。
- **誠實標註**：屬「模擬重建」（以現行邏輯回算歷史，含未來函數風險），輸出 `method` 欄位明示。

### 3.2 呈現
- `/api/stats` 附 `honestStats`（讀 JSON，缺檔回 null）。
- `/stats` 頁新增「誠實統計」卡（高分群三情境並排、紅綠盤分段、判讀提示），置於 KPI 之後、所有示範圖卡之前。

## 6. 已知限制（誠實聲明，也是後續工作）

- honest_stats 屬**模擬重建**（含未來函數風險）；真正「前向定格」由 track-record Phase 2 解決（§7）。
- 樣本仍偏多頭區間、量少；CI 寬，紅/綠盤分段樣本更少 — UI 已示警。
- 「隔日開盤價」不保證可成交，低流動性實際滑價可能高於保守情境。
- `run_backtest.py` 的評分漂移（缺兩訊號）**仍存在於舊回測**（標 legacy 待退役）；誠實統計不受其影響。
- honest_stats 目前手動執行更新；尚未排進 daily Action（見 §8）。

## 7. 與 forward-track-record spec 的關係

該 spec 的技術設計（focus-picks 抽取、snapshot/grade/aggregate、不可變快照）**全部保留有效**，但定框改變：
- 原：「證明選股會賺」的戰績 → 新：「**透明可稽核的隔日行為資料集**」（教育旗艦的資料底座）。
- 頭條指標從「勝率」改為「含成本中位數＋分布」（與誠實統計同口徑）。
- 優先序：列為 Phase 2（本次重定位為 Phase 1）。
